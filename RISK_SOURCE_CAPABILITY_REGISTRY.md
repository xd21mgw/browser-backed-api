# Risk Source Capability Registry

This is the team-facing capability menu for the local browser-backed risk
platform access service. It is organized by risk evidence domain, not by HAR
file or platform implementation detail.

The registry describes evidence capabilities that Agent, Skill, or local scripts
may call through fixed local actions on `127.0.0.1`. It is not a list of every
observed platform request. Frontend telemetry, static assets, logging SDKs,
fingerprinting, menu/config probes, and arbitrary URL fetches are not open
capabilities.

## Evidence Domains

| evidence_domain | Scope |
| --- | --- |
| 用户域 | User profile, account state, registration, real-name/profile shape, historical risk hints. |
| 设备域 | Device graph, device risk labels, device activity, user-device relation. |
| 行为域 | Login, active usage, publish/comment/follow/view behavior when a bounded evidence source exists. |
| 策略域 | Strategy hit events, event details, features, attribution hints, policy tree lookup. |
| 内容域 | Video, live, photo/text content, diversion, content-risk evidence. |
| 社交域 | Private message, related users, relation chain evidence. |
| 监控域 | Grafana/dashboard/trend evidence. |

## Stable Default Capabilities

These capabilities are stable enough for general internal risk review. They are
the default source actions an Agent may consider after identifying the evidence
domain.

### track_analysis_summary

| Field | Value |
| --- | --- |
| capability_name | User activity/profile/device evidence summary |
| evidence_domain | 用户域 / 设备域 / 行为域 |
| user_facing_description | Summarizes profile shape, recent activity/use-duration, device list, and latest activity availability for a typed user or device. |
| action_name | `track_analysis_summary` |

| current_status | `stable` |
| default_for_general_review | yes |
| explicit_trigger_required | no |

## Promotion Rule

A new action can be promoted from `inventory_pending` or `contract_ready` only
after all of the following exist:

- Source contract with evidence purpose and non-goals.
- Fixed origin and same-origin relative path in registry/config.
- Typed params with forbidden caller surface enforced.
- `source_card` and `source_quality` in every result.
- Redaction policy for raw bodies, records, labels, logs, PII, and risk entity identifiers.
- Mock tests for success, no-data, auth/blocked/error, forbidden input, and redaction.
- Live smoke evidence with no credential material output.

No promotion should add arbitrary URL access, automatic disposal, permission
bypass, automatic DataAgent/Hive calls, or final risk classification.
