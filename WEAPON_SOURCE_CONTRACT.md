# Weapon Source Contract

## Source Identity

- source_name: `weapon_inventory`
- access_method: `browser_backed_api_service`
- fixed action endpoint: `POST http://127.0.0.1:8787/actions/weapon_inventory`
- origin: `https://weapon-platform.corp.kuaishou.com`
- runtime boundary: the browser-backed API service owns the persistent Chrome context; Dennis or other callers only call the fixed local action endpoint.

## Supported APIs

The action supports exactly these same-origin Weapon APIs:

- `GET /apiv2/graphData`
- `GET /apiv2/riskData`

The caller cannot choose arbitrary URLs, paths, origins, headers, cookies, tokens, sessions, secrets, or raw request bodies.

## `graphData` Contract

`graphData` is the primary source call. It discovers user/device graph shape and, when device IDs are present, provides internal chaining input for `riskData`.

Fixed path:

- `GET /apiv2/graphData`

Typed params:

- exactly one of `user_id` or `device_id`
- `product=KUAISHOU`
- `productName=KUAISHOU`
- `searchLevel=2`

Mapping:

- `user_id` input:
  - `groupValue=<user_id>`
  - `groupKey=USER_ID`
  - `dimKey=DEVICE_ID`
- `device_id` input:
  - `groupValue=<device_id>`
  - `groupKey=DEVICE_ID`
  - `dimKey=USER_ID`

`device_id` values must preserve the full platform prefix such as `ANDROID_` or `IOS_`.

## `riskData` Contract

`riskData` is a chained source call. It runs only after `graphData` returns one or more device IDs with full `ANDROID_` or `IOS_` prefixes.

Fixed path:

- `GET /apiv2/riskData`

Typed params generated by the service:

- `product=KUAISHOU`
- `deviceIds=<device_id from graphData internal chaining>`

The `deviceIds` param is internal chaining state. It is not accepted from the caller as raw path, raw query, or raw body input, and it is not displayed in the final source output.

## `graphData` Output Summary

The action returns shape-only graph summary fields:

- `graph_status`
- `graph_api_code`
- `graph_api_msg`
- `pointInfoMap_present`
- `pointInfoMap_count`
- `relationEdgeList_present`
- `relationEdgeList_count`
- `related_device_count`
- `related_user_count`
- `masked_device_id_sample`
- `raw_device_ids_for_internal_chaining_count`
- `no_data_not_risk_exclusion=true`

Parsing rules:

- `pointInfoMap` is an object/map.
- `relationEdgeList` is an array.
- Device IDs are recognized only when they preserve full `ANDROID_` or `IOS_` prefixes.
- Pure numeric graph keys are treated as probable user IDs, not device IDs.
- Empty `pointInfoMap` and empty `relationEdgeList` are `no_data` or `completed_no_data`, not platform failure and not no-risk counterevidence.

## `riskData` Output Summary

The action returns shape-only risk summary fields:

- `riskData_status`
- `risk_item_count`
- `risk_label_summary`
- `risk_label_count`
- `risk_group_names_observed`
- `readable_label_sample`
- `originalLog_key_summary`
- `userLevel_observed`
- `no_data_not_risk_exclusion=true`

Parsing rules:

- `data` may be a list; the parser treats list entries as risk items.
- Risk item fields may include `deviceId`, `productName`, `labelInfo`, `originalLog`, and `userLevel`.
- `labelInfo` is summarized by counts, group names, and readable label samples only.
- `originalLog` is summarized by key presence and key counts only.
- `userLevel` values are reported as supporting source evidence.

## Redaction Rules

- Raw device IDs are used only as internal chaining input from `graphData` to `riskData`.
- Display output may include only masked device samples such as `[masked_device_id:length=N]`.
- Raw `labelInfo` is not output.
- Raw `originalLog` is not output.
- Raw upstream full body is not output.
- `source_card.path` uses a display path that does not expose caller-provided `user_id` or `device_id`.
- Sensitive-looking JSON key names are redacted in shape summaries.

## Status And Error Semantics

- `completed`: `graphData` succeeded and graph summary was produced. `riskData` may also have completed.
- `completed_no_data`: `graphData` succeeded but returned no useful graph nodes or edges. This is not no-risk counterevidence.
- `no_data`: source returned no useful data. This is not no-risk counterevidence.
- `not_executed_missing_device_id`: `graphData` returned no `ANDROID_` or `IOS_` device IDs, so `riskData` was not executed.
- `risk_partial_failed`: at least one `riskData` chained call failed or could not be parsed. This does not overwrite graph success; preserve partial `source_quality`.
- `auth_failed`: the browser page or fetch result indicates SSO/login/auth redirect. This is a source access outcome, not a runtime failure.
- `parse_error`: the response could not be parsed as expected JSON and did not match the SSO/login classification.
- `platform_error`: Weapon returned an HTTP or API-level error. Preserve `source_card` and `source_quality`; do not treat it as Dennis runtime failure.
- `network_error`: local browser fetch failed before a valid platform response was summarized. Preserve `source_card` and `source_quality`.
- `parameter_error`: typed params are missing or invalid. No platform call is made.

`no_data`, `completed_no_data`, `not_executed_missing_device_id`, `auth_failed`, `blocked`, `timeout`, `network_error`, and `platform_error` must not be used as no-risk counterevidence.

## Security Boundary

- The service does not read Chrome cookie storage.
- The service does not call cookie, token, session, or header inspection APIs.
- The service does not output cookies, tokens, sessions, request headers, raw device IDs, raw `labelInfo`, raw `originalLog`, or raw upstream full bodies.
- The service accepts only fixed action names and fixed same-origin relative paths.
- The service rejects caller-supplied URL, path, origin, header, cookie, token, session, secret, and raw body material.

## Dennis Evidence Card Guidance

- Use `graphData` summary as device relation evidence.
- Use `riskData` summary as device risk label evidence.
- Treat `userLevel_observed` and risk group names as supporting evidence only.
- Do not use Weapon output alone as a final account risk conclusion.
- Include `source_card` and `source_quality` for every source result, including failures and partial risk outcomes.
- Treat `no_data`, `completed_no_data`, `not_executed_missing_device_id`, `risk_partial_failed`, `auth_failed`, `blocked`, `timeout`, `network_error`, and `platform_error` as source quality/completion states, not as no-risk counterevidence.
